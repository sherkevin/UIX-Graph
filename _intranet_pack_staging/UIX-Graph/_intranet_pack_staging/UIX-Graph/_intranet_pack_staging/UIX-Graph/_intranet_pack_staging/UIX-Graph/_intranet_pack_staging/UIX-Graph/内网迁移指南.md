# UIX 内网迁移指南
## SXEE-LITHO-RCA 光刻机拒片根因分析系统

---

## 一、双击启动（Windows）

将整个压缩包解压到任意目录，例如 `C:\UIX\`，然后：

**双击 `启动UIX.bat`** → 自动弹出启动器窗口 → 点击 **▶ 启动** → 浏览器自动打开。

> 首次启动会自动安装 Python 依赖（需联网约 1-2 分钟）。内网无法联网时，请提前在联网电脑运行一次安装依赖，或由管理员提前执行 `pip install -r src\backend\requirements.txt`。

---

## 二、连接内网真实数据库（只改这一个文件）

打开 `config/connections.json`，修改 `prod` 节的数据库地址：

```json
"prod": {
    "mysql": {
        "host":     "改成内网 MySQL 的 IP 或主机名",
        "port":     3306,
        "username": "改成数据库用户名",
        "password": "改成数据库密码",
        "dbname":   "datacenter"
    },
    "clickhouse": {
        "host":     "改成内网 ClickHouse 的 IP",
        "port":     8123,
        "username": "改成用户名（可为空）",
        "password": "改成密码（可为空）",
        "dbname":   "las"
    }
}
```

改完之后，打开 `src/backend/.env`，将第一行改为：

```
APP_ENV=prod
```

然后重启即可，**其余代码文件一行不动**。

## 三、关于 ClickHouse 数据源

| 场景 | 操作 |
|------|------|
| ClickHouse 暂时不可用 | `src/backend/.env` 中设置 `METRIC_SOURCE_MODE=mock_allowed`（默认），系统将用模拟值自动降级，不报错 |
| ClickHouse 已接通，必须用真实数据 | 设置 `METRIC_SOURCE_MODE=real`，ClickHouse 不可达时接口会返回错误提示 |

## 四、前端无需 Node.js（已预编译）

压缩包内已包含预编译前端 (`src/frontend/dist/`)，Windows 机器上**无需安装 Node.js**。  
启动器直接用 Python 内置 HTTP 服务器承载前端页面，同时将 `/api/*` 请求转发到后端。

## 五、环境要求

| 软件 | 最低版本 | 说明 |
|------|----------|------|
| Python | 3.9+ | 安装时勾选"Add to PATH"和"tcl/tk and IDLE" |
| MySQL | 5.7 / 8.0 | 内网已有，只需在 `connections.json` 填入连接信息 |
| ClickHouse | 22+ | 可选，不接入时系统以模拟值运行 |

## 六、端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8000 | FastAPI，内网机器防火墙需允许 |

## 七、常见问题

**Q: 双击 `启动UIX.bat` 闪退**  
A: 右键 → "以管理员身份运行"，或检查 Python 是否已加入 PATH。

**Q: 浏览器打开后显示"无法连接服务器"**  
A: 等待启动器状态变成绿色"运行中"后再刷新浏览器，后端启动需要约 5 秒。

**Q: 机台没有数据**  
A: 确认 `config/connections.json` 中数据库连接地址正确，且数据库中 `lo_batch_equipment_performance` 表有数据。

**Q: Windows 中文路径乱码**  
A: `启动UIX.bat` 已设置 `chcp 65001`（UTF-8 编码），Python 所有文件均为 UTF-8。若仍有乱码，请确认系统区域设置为"Beta版：使用 Unicode UTF-8"。
